┌─────────────────────────────────────────────────────────────────────────────┐
│ TYPICAL WORKFLOW                                                            │
└─────────────────────────────────────────────────────────────────────────────┘

  1. Edit code
  2. run.bat                                    (test changes)
  3. Repeat steps 1-2 until working
  4. build.bat                                  (verify tests pass)
  5. Git commit & push

  WHEN RELEASING:
  6. Update version in tf-engine-installer.nsi
  7. build-installer.bat
  8. Test TFEngine-Setup-1.0.0.exe
  9. Create GitHub release
  10. Upload installer + .sha256 hash

┌─────────────────────────────────────────────────────────────────────────────┐
│ TROUBLESHOOTING                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

  Tests failing?                build.bat shows errors → fix tests

  Need to build anyway?         quick-build.bat (skips tests)

  NSIS not found?               Install from: nsis.sourceforge.io/Download
                                or edit NSIS_PATH in build-installer.bat

  Build behaving weird?         clean.bat (fresh start)

  Exe file huge (40 MB)?        This is normal! Go includes full runtime.
